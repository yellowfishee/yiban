> 风格方向：东方神秘古风 | 改进范围：全流程（蓄力 → 揭示 → 呈现）

## 一、现状问题诊断

| 问题 | 具体表现 | 严重程度 |
|------|----------|----------|
| 无仪式感铺垫 | 卡片直接展示，点击即翻，没有任何"蓄力"过程 | 🔴 高 |
| 动画过于简单 | 仅 0.6s 的 `rotateY(180deg)`，缓动曲线平淡 | 🔴 高 |
| 背面设计单调 | 纯深色渐变 + 阿拉伯数字序号，毫无东方神秘元素 | 🔴 高 |

## 二、核心设计理念

借鉴**塔罗牌占卜**和**开光仪式**的体验节奏，将翻牌拆分为三个阶段：

```
[第一阶段：蓄力]  →  [第二阶段：揭示]  →  [第三阶段：呈现]
  长按卡片            松手完成翻转         内容逐字浮现
  光晕聚拢            粒子迸发 + 墨迹散开   古风装饰框淡入
  八卦符文旋转        3D 翻转加速→减速      神兽水印浮现
```

---

## 三、三阶段交互详细设计

### 3.1 阶段一：蓄力（长按 0.8~1.5s）

| 效果 | 实现方式 |
|------|----------|
| 卡片微微浮起 + 放大 | `transform: scale(1.03) translateY(-4px)` |
| 周围光晕从无到有聚拢 | CSS `box-shadow` 动画 + 伪元素 `radial-gradient` |
| 八卦符文在卡片表面缓缓旋转 | CSS `@keyframes rotate` + Unicode 符文 `☰ ☷ ☲ ☵` |
| 低频震动反馈 | `Taro.vibrateShort({ type: 'medium' })` |
| 蓄力进度环 | CSS `conic-gradient` 从 0° 到 360° |

**交互逻辑**：
- 用户按下卡片 → 进入蓄力状态
- 持续按住 0.8s 后蓄力完成 → 卡片边缘发出微光提示"可以松手了"
- 蓄力期间松手 → 取消蓄力，卡片恢复原状
- 蓄力完成后松手 → 自动进入第二阶段（揭示）

### 3.2 阶段二：揭示（翻转动画 1.2s）

| 效果 | 实现方式 |
|------|----------|
| 先快后慢的翻转 | `cubic-bezier(0.22, 1, 0.36, 1)`（减速曲线） |
| 翻转到 90° 时停顿 200ms | 分两段动画：0→90° (0.5s) → 停顿 → 90°→180° (0.5s) |
| 停顿瞬间边缘发光 | `box-shadow` 金色光晕 `0 0 30px rgba(201, 168, 76, 0.6)` |
| 粒子从卡片边缘迸发 | CSS 伪元素 + 多个 `@keyframes` 模拟 6-8 个粒子飞散 |
| 墨迹/水波纹从中心扩散 | CSS `radial-gradient` 动画 |

**动画时序**：
```
0ms ──────── 500ms ──── 700ms ──────── 1200ms
│  翻转 0→90°  │ 停顿 │ 翻转 90→180°  │
│  (快速)      │ 发光 │  (减速)        │
│              │ 粒子 │                │
```

### 3.3 阶段三：呈现（内容浮现 1.5s）

| 效果 | 实现方式 |
|------|----------|
| 文字逐字浮现（非打字机） | 每个字从 `opacity:0; translateY(8px)` 渐入，间隔 50ms |
| 场景标签先出现 | 标签 0.3s 淡入 → 内容开始逐字浮现 |
| 神兽水印在背景浮现 | `opacity: 0→0.06`，大号卦象文字作为水印 |
| 金色边框描边动画 | CSS `background` 渐变描边 + `background-size` 动画 |
| 轻柔震动 | `Taro.vibrateShort({ type: 'light' })` |

## 四、视觉设计

### 4.1 背面重设计：东方神秘古风

```
┌──────────────────────────────┐
│  ☰  ☷  ☲  ☵  （八卦符文环绕） │
│  ┌────────────────────────┐  │
│  │                        │  │
│  │         ┌──────┐       │  │
│  │         │  壹   │       │  │  ← 古风大字替代阿拉伯数字
│  │         └──────┘       │  │
│  │                        │  │
│  │        今日适合         │  │  ← 竖排或居中
│  │                        │  │
│  │    ── 长按揭开指引 ──   │  │  ← 古风分隔线
│  │                        │  │
│  └────────────────────────┘  │
│  ☳  ☴  ☶  ☱  （八卦符文环绕） │
└──────────────────────────────┘
```

**视觉元素清单**：

| 元素 | 设计 |
|------|------|
| 背景 | 深墨色渐变 `#1A1A2E → #0F0F1A` + 宣纸纹理（CSS noise pattern） |
| 边框 | 金色细线 `1px solid rgba(201, 168, 76, 0.3)` |
| 四角 | 中式角花装饰（CSS border 或伪元素） |
| 序号 | 古风汉字：壹、贰、叁、肆、伍、陆 |
| 符文 | 八卦 Unicode 在四角缓慢旋转 `☰ ☷ ☲ ☵ ☳ ☴ ☶ ☱` |
| 标题字体 | `STSong, SimSun, serif` |
| 装饰线 | `━━━ ✦ ━━━` 古风分隔符 |

### 4.2 正面重设计

```
┌──────────────────────────────┐

│                              │
```

**视觉元素清单**：

| 元素 | 设计 |
|------|------|

- **实现**：每个字从 `opacity:0; translateY(8px)` 渐入，间隔 50ms
- **依赖**：无

### 子任务 3：新建粒子光效组件 MysticParticles

- **目的**：翻牌揭示时的视觉特效（粒子迸发 + 墨迹扩散）
- **文件**：`packages/app/src/components/hexagram/MysticParticles.tsx` + `MysticParticles.scss`
- **实现**：CSS 伪元素 + 多个 `@keyframes` 模拟 6-8 个粒子飞散
- **依赖**：无

### 子任务 4：重写翻牌状态逻辑

- **目的**：支持三阶段交互（蓄力 → 揭示 → 呈现）
- **文件**：`packages/app/src/pages/home/index.tsx`
- **改动**：
  - 新增状态：`flipPhase: 'idle' | 'charging' | 'revealing' | 'revealed'`
  - 新增状态：`chargeProgress: number`（蓄力进度 0-100）
  - 长按事件：`onTouchStart` → 开始蓄力，`onTouchEnd` → 判断是否完成蓄力
  - 翻转触发：蓄力完成松手 → 分两段执行翻转动画
  - 内容呈现：翻转完成后 → 触发 RevealText 组件
  - 震动反馈：蓄力开始/完成/翻转完成时触发
- **依赖**：子任务 2、3

### 子任务 5：重写翻牌 SCSS

- **目的**：东方古风视觉 + 三阶段动画
- **文件**：`packages/app/src/pages/home/index.scss`
- **改动**：
  - 背面样式：深墨色 + 宣纸纹理 + 金色边框 + 八卦符文 + 古风序号
  - 正面样式：金色标签 + 神兽水印 + 描边动画
  - 蓄力动画：`@keyframes charge`（光晕聚拢 + 进度环）
  - 翻转动画：分两段 `@keyframes flipFirstHalf` + `flipSecondHalf`
  - 粒子动画：`@keyframes particleBurst`（6-8 个粒子飞散）
  - 轮播切换：收牌 → 滑动 → 出牌过渡动画
  - 控件样式：古风箭头 + 竖排圆点 + 古风计数
- **依赖**：无

### 子任务 6：集成测试与调优

- **目的**：验证动画流畅度和整体体验
- **验证项**：

  - 6 张卡片均可翻开
  - 轮播切换流畅
  - 震动反馈正常触发
- **依赖**：子任务 1-5

## 八、风险与注意事项

| 风险 | 应对策略 |
|------|----------|
| 小程序不支持部分 CSS 特性（如 `conic-gradient`） | 使用 `Taro.canIUse()` 检测，降级为简单进度条 |
| 长按事件与小程序默认长按菜单冲突 | 使用 `catchTouchMove` 阻止冒泡 |
| 动画性能问题（低端机卡顿） | 使用 `transform` + `opacity` 仅触发 GPU 合成层 |
| `Taro.vibrateShort` 在部分机型不支持 | try-catch 包裹，静默失败 |
| 粒子动画在 H5 和小程序表现不一致 | 使用 Taro 通用 API，避免平台特有 CSS |
