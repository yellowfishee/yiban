# 梅花易数每日打卡设计方案

> 日期：2026-04-02
> 状态：已批准实施

---

## 一、目标

将每日打卡签到与梅花易数起卦结合，以打卡时间戳为种子计算今日主卦。打卡即领养，有动爻时额外展示变卦/互卦作为参考。

---

## 二、起卦算法

### 2.1 纯时间戳随机算法

```
1. 取时间戳末 2 位 → 记作 A
2. 生成随机三位小数（0.000-0.999）→ 记作 B
3. 乘积 = A × B
4. 乘积小数部分 × 1000，取前三位 → 种子数 N（如 27.104 → N = 104）

上卦  = N % 8          → 0=乾, 1=坤, 2=屯, 3=蒙, 4=需, 5=讼, 6=师, 7=比
下卦  = (N + 年份%8) % 8
动爻  = (N % 6) + 1    → 1-6，第几爻动
```

**八卦对应表（0-7）：**
| 余数 | 卦名 | ID |
|------|------|-----|
| 0 | 乾 | qian |
| 1 | 坤 | kun |
| 2 | 屯 | zhun |
| 3 | 蒙 | meng |
| 4 | 需 | xu |
| 5 | 讼 | song |
| 6 | 师 | shi |
| 7 | 比 | bi |

**动爻判定：**
- 动爻 > 0：有动爻，产生变卦（该爻阴阳互换）和互卦（上卦不变，下卦变下卦的三四爻）
- 动爻 = 0 时，视作动爻 = 6（老阳/老阴变爻规则）

### 2.2 变卦计算规则

动爻阴阳互换：
- 阳爻（—）动 → 变阴爻（--）
- 阴爻（--）动 → 变阳爻（—）

变卦 = 主卦该爻变化后的新卦

### 2.3 互卦计算规则

互卦规则（简化版）：取主卦的二三四五爻：
- 互卦上卦 = 主卦三四爻组成的卦
- 互卦下卦 = 主卦二三四爻组成的卦

**注意**：当前 MVP 数据仅含 12 卦（乾坤屯蒙需讼师比大有蛊临观），梅花算法可产生 8 卦范围内结果，保证数据可匹配。后续扩展 64 卦时算法需相应调整。

## 三、数据结构

### 3.1 新增 Storage Keys

| Key | 类型 | 说明 |
|-----|------|------|
| `yiban_checkin_timestamp` | string | 今日打卡时间戳（毫秒）|
| `yiban_main_hexagram_id` | string | 主卦 ID |
| `yiban_has_moving_line` | string | 是否有动爻（"true"/"false"）|
| `yiban_moving_line` | string | 动爻数（1-6）|
| `yiban_bian_hexagram_id` | string | 变卦 ID（有动爻时）|
| `yiban_hu_hexagram_id` | string | 互卦 ID（有动爻时）|

### 3.2 新增 TypeScript 类型

```typescript
interface MeihuaResult {
  mainHexagram: Hexagram;       // 主卦
  upperGua: number;             // 上卦（0-7）
  lowerGua: number;             // 下卦（0-7）
  movingLine: number;           // 动爻（1-6）
  hasMovingLine: boolean;       // 是否有动爻
  bianHexagram?: Hexagram;      // 变卦（有动爻时）
  huHexagram?: Hexagram;        // 互卦（有动爻时）
}
```

## 四、Context 逻辑变更

### InspirationContext

**loadToday 逻辑：**

```
if (今日已打卡) {
  → 读取缓存的主卦 ID，恢复展示（无起卦动画）
} else {
  → 执行梅花起卦
  → 展示起卦动画（2-3秒）
  → 主卦存入 storage，打卡即领养
  → 若有动爻，同时存入变卦/互卦 ID
}
```

**adopt 逻辑不变**（已打卡状态下 adopt 无操作），领养行为在打卡时自动完成。

## 五、页面交互设计

### 5.1 首次加载（今日未打卡）

```
→ 显示起卦动画（2-3秒淡入）
→ 展示起卦过程：
    - 上卦：年数字 + 上卦名
    - 下卦：时数字 + 下卦名
    - 动爻：第几爻动
→ 展示主卦大卡片（可浮动动画）
→ 若有动爻 → 下方同时展示变卦/互卦小卡（灰色半透明，不可点击）
→ "打卡领养" 按钮（点击后完成打卡 + 领养，变为已打卡状态）
```

### 5.2 再次加载（今日已打卡）

```
→ 直接展示主卦（无动画）
→ "今日已打卡，明日再来" 文字
→ 打卡按钮区域显示已打卡状态
```
