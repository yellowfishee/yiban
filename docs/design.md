# 易伴·卦象神兽 首页设计文档

> 本文档记录首页（今日 Tab）的设计决策与实现细节。

## 一、设计目标

打造沉浸式的"遇见神兽"体验，强调神秘感与仪式感。通过塔罗牌式的翻牌交互，让用户在探索中获取每日的智能指引。

## 三、页面状态

### 3.1 未打卡状态（初始状态）

用户打开首页时看到：

- **标题**：今日灵感
- **欢迎区块**：
  - 太级符号 ☯
  - "梅花易数起卦"
  - "以此刻为锚，感而遂通天下之故"
- **引导文案**：解释梅花起卦的作用
- **主按钮**：梅花起卦 · 打卡领养（朱红色）

用户点击按钮后：
1. 若未登录，自动拉起微信登录
2. 完成登录后执行打卡
3. 展示今日卦象与灵感

### 3.2 已打卡状态

- **神兽卡片**：展示今日卦象神兽（大字名称 + 图标 + 浮动动画）
- **灵感絮语**：核心文案，根据心境生成
- **梅花易数**：上卦、下卦、动爻信息
- **神兽建议**：6 张翻牌（见本文档第四章）
- **底部提示**：今日已打卡，明日再来遇见新伙伴吧
- **分享按钮**：H5 环境显示，唤起系统分享

## 四、神兽建议：翻牌交互

### 4.1 设计理念

参考塔罗牌翻牌的仪式感。用户可以自主选择揭开哪张牌，营造"探索命运"的神秘氛围。

### 4.2 交互逻辑

| 状态 | 视觉 | 交互 |
|------|------|------|
| 未翻开 | 深色卡片 + 序号 + 场景名 + "点击揭开" | 可点击 |
| 已翻开 | 浅色卡片 + 场景名 + 建议内容 | 再次点击翻回背面 |
| 未解锁 | 暂时不做区分，所有卡片可点击 | - |

### 4.3 卡片布局

- **网格**：2 列 x 3 行
- **卡片尺寸**：高度 200px，宽度自适应
- **间距**：20px 间距

### 4.4 视觉效果

**背面（默认）**：
```
┌─────────────────┐
│              01 │  ← 大号淡色序号
│                 │
│    处事建议      │  ← 白色标签
│                 │
│    点击揭开      │  ← 淡色提示
└─────────────────┘
```
- 背景：深灰渐变 `#2D3748 → #1A202C`
- 字号：序号 56px，标签 28px，提示 20px
- 动画：3D rotateY 翻转，600ms

**正面（翻开）**：
```
┌─────────────────┐
│   处 事 建 议    │  ← 小号淡色标签
│                 │
│  保持谦逊与专注  │  ← 内容文案
│  今日宜静心思考  │
└─────────────────┘
```
- 背景：米白渐变 `#FDFCF9 → #F5F0E8`
- 字号：标签 22px，内容 26px

## 五、隐藏功能：今日心境

### 5.1 决策背景

最初设计了"今日心境"提示区，让用户选择工作、情感等心境后生成对应灵感。但经产品讨论认为：

- 场景建议（翻牌）已涵盖心境相关内容
- 多余的心境选择会增加用户认知负担
- 打卡后的体验应该更沉浸，减少选择

### 5.2 当前状态

- **MoodSelector 组件**：保留但不在首页渲染
- **selectedMood 状态**：仍用于生成灵感絮语（默 认为 work）
- **后续扩展**：若需恢复心境选择，可将 MoodSelector 以横向滑动条形式重新引入

## 七、技术实现

### 7.1 文件结构

```
packages/app/src/pages/home/
├── index.tsx      # 页面逻辑
└── index.scss     # 样式
```

### 7.2 状态管理

```typescript
// 已打卡状态下的翻牌状态
const [flippedCards, setFlippedCards] = useState<Set<AgentScene>>(new Set());

const toggleFlip = (scene: AgentScene) => {
  setFlippedCards(prev => {
    const next = new Set(prev);
    if (next.has(scene)) {
      next.delete(scene);
    } else {
      next.add(scene);
    }
    return next;
  });
};
```

### 7.3 条件渲染

- 未打卡：显示欢迎区 + 打卡按钮
- 已打卡：显示神兽卡片 + 灵感絮语 + 翻牌 + 分享按钮
- 加载中：显示 spinner + "正在遇见神兽..."
- 错误：显示错误文案 + 重试按钮

## 八、后续扩展

### 8.1 神兽图片

上线后计划为每个卦象神兽配置专属图片。届时：

- HexagramCard 组件将展示图片而非纯文字
- 神兽图片存放于 `src/assets/beasts/` 目录
- 需考虑图片加载占位与懒加载

### 8.2 翻牌解锁机制

后续若上线激励视频广告，锁定状态的卡片可设计为：

- 点击后弹出激励视频
- 看完视频解锁该场景
- 解锁后自动翻开显示内容

### 8.3 心境选择回归

若需恢复心境选择，可考虑：

- 将 MoodSelector 改为横向滑动的小圆点选择器
- 选择后重新生成灵感絮语
- 场景建议内容随心境变化
